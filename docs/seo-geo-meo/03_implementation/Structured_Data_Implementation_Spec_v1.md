# Structured Data Implementation Spec v1

## 概要
株式会社清蓮のコーポレートサイトにおける構造化データの実装方針を定義する。
SEO・GEO・MEO の整合を重視し、検索エンジン、AI引用面、ローカル検索面に対して、表示内容と完全一致する JSON-LD のみを実装対象とする。

## 目的
LocalBusiness、FAQPage、BreadcrumbList などの構造化データ実装方針を整理し、実装判断の基準を定義する。

## 対象範囲
- JSON-LDの対象タイプ
- 出し分け方針
- 実装・検証・保守の運用基準

## 対象スキーマ
### 優先実装スキーマ

#### 1. `Organization`
- 対象ページ: トップページ、全ページ共通レイアウト
- 採用理由:
  - コーポレートサイト全体の運営主体を明示できる
  - SEO と GEO の基礎情報として機能する
  - LocalBusiness よりも確認未了項目が少なく、安全に先行実装しやすい
- 実装方針:
  - `name` は `株式会社清蓮`
  - `address` と `telephone` は NAP マスターの確定値のみ使用
  - `url` は `https://seiren-inc.co.jp/` を使用する
  - `logo` は `https://seiren-inc.co.jp/icon.png` を使用する

#### 2. `WebSite`
- 対象ページ: トップページ
- 採用理由:
  - サイト全体の主体と URL を補強できる
  - ブランド名検索時の整合性向上に寄与する
- 実装方針:
  - `name` は `株式会社清蓮`
  - `url` は `https://seiren-inc.co.jp/` を使用する
  - サイト内検索が実装されていない間は `SearchAction` を追加しない

#### 3. `BreadcrumbList`
- 対象ページ: 下層ページ全般
- 採用理由:
  - 検索結果・AI引用面・サイト内情報設計の整合に有効
  - 現行実装との親和性が高い
- 実装方針:
  - UIに表示されるパンくずと完全一致させる
  - 各 `item` URL は `https://seiren-inc.co.jp/` ベースで生成する

#### 4. `FAQPage`
- 対象ページ: FAQ セクションを持つページ、または将来のFAQ専用ページ
- 採用理由:
  - FAQ がページ上に実在する場合、SEO と GEO の両面で引用性を高めやすい
- 実装条件:
  - ページ上に質問と回答が実際に表示されていること
  - 会社情報やサービス名が NAP マスターと一致していること
  - 推測で作成したQ&Aを実装しないこと

### その他候補
- `LocalBusiness`
  - ローカル検索最適化に有効であり、営業時間、郵便番号、GBP整合確認が完了したため採用対象とする
  - `url` はコーポレートサイトを表す `https://seiren-inc.co.jp/` を使用する
  - 散骨クルーズ事業サイト `https://seiren.ne.jp/` は必要に応じて別導線または補足情報として扱う
- `WebPage`
  - 必須ではないが、ページテンプレート統一時に追加検討可
- `Service`
  - サービス個別ページが分離した時点で検討する
- `ContactPage`
  - 専用ページ要件と表示内容が整理された後に検討する

## 実装方針
- 形式は JSON-LD のみを使用する。
- 1ページに複数タイプを入れる場合でも、ページの主目的と表示内容に整合するものだけを実装する。
- 表示されていない情報を構造化データだけに先行記載しない。
- 会社固有情報は、確認済み情報と未確認情報を分けて扱う。
- 確認未了の会社情報は、仮値・推定値・他ページ参照値のまま掲載しない。

### 最適構成の推奨
- トップページ:
  - `Organization`
  - `WebSite`
  - `LocalBusiness`
- 下層ページ:
  - `BreadcrumbList`
- FAQ 掲載ページ:
  - `FAQPage`

### LocalBusiness の採用条件
- 以下がすべて確認済みであること
- 正式URL
- 郵便番号
- 営業時間
- GBP登録名
- GBP登録住所
- GBP登録電話番号
- GBP登録営業時間
- 現在は上記確認が完了しているため、`Organization` と併用して `LocalBusiness` を実装してよい

### 会社情報の記載ルール
- 記載可:
  - `株式会社清蓮`
  - `〒244-0003 神奈川県横浜市戸塚区戸塚町4170 高橋ビル1F`
  - `神奈川県横浜市戸塚区戸塚町4170 高橋ビル1F`
  - `045-881-9952`
  - `月曜〜日曜 9:00〜17:00`
  - `Mo-Su 09:00-17:00`
- 記載保留:
  - `sameAs` 対象URL
  - 緯度経度

### 現時点で実装しない項目
- 未確認の `sameAs`
- 未確認の `geo`
- サイトに存在しない FAQ やレビュー情報

## データソース定義
- 最優先ソース:
  - `docs/seo-geo-meo/03_implementation/NAP_Master_Definition_v1.md`
- 確認ソース:
  - 公開サイトの表示内容
  - Googleビジネスプロフィールの確定登録内容
  - 会社の正式案内情報
- 補助ソース:
  - 既存コード内の実装値
- 補助ソースはそのまま採用しない。必ず NAP マスターまたは担当確認で照合する。

| 項目 | 現状 | 対応 |

| `sameAs` 対象URL | 未整理 | 公式SNS・外部媒体確認後に掲載 |
| 緯度経度 | 未整理 | 公式住所基準で確認後に掲載 |

## 検証方法
- 実装前チェック
  - H1、title、本文、会社情報ブロック、構造化データの `name` が一致していること
  - NAPマスターと一致していること
  - 未確認項目が JSON-LD に含まれていないこと
- 実装後チェック
  - Rich Results Test でエラー・重大警告がないこと
  - Schema Markup Validator で構文エラーがないこと
  - ページ表示内容と JSON-LD が完全一致していること
  - パンくずの UI と `BreadcrumbList` が一致していること
- 公開前チェック
  - 会社固有値は担当確認済みか
  - GBP に記載された NAP と矛盾しないか
  - `LocalBusiness` の `openingHours` が `Mo-Su 09:00-17:00` で統一されているか
  - FAQ はページ上に実在するか

## 注意事項
- 誤情報を一切載せないことを最優先とする。
- 会社固有情報は、ユーザー確認または NAP マスター確定前に公開実装しない。
- `LocalBusiness` は採用可能になったが、`url` はコーポレートサイト `https://seiren-inc.co.jp/` を使用し、散骨クルーズ事業サイト `https://seiren.ne.jp/` と混同しない。
- 構造化データは「最適そうな値」を埋めるのではなく、「確認済みの値だけ」を埋める。

## 確認依頼が必要な項目
- `sameAs` として掲載する公式SNSや外部プロフィールURLはあるか
- 緯度経度を明示的に管理するか

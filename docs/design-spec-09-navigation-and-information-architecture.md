# 清蓮コーポレートサイト デザイン設計書 09

## ナビゲーション / 情報設計（IA）統合設計

---

# 1. 目的

迷わない構造・理解しやすい階層・静かな導線を統一するためのIA規格。

# 2. 情報階層設計

## 第1階層（グローバルナビ）

- 事業紹介 / 清蓮の強み / 提携・取引先 / 会社情報 / お問い合わせ
- 最大6項目。増やさない。

## 第2階層

### 事業紹介配下（専門サイトへリンク）

海洋散骨 / 遺骨サービス / お墓じまい / 遺骨ダイヤモンド / 終活コンシェルジュ / 手元供養 / 国際遺骨搬送 / お墓探しナビ

### 会社情報配下

会社概要 / 代表挨拶 / 所在地 / アクセス

# 3. ナビゲーション設計

## PC ヘッダー

- 横並び / 背景：--bg / 下線：1px --line / 高さ: 80px前後
- hover：brand-primaryへ色変化 + 下線が滑らかに表示（transition）

## モバイルナビ

- ハンバーガー / フルスクリーン展開 / 背景：--bg
- 開閉アニメ 300ms以内

# 4. パンくず設計

全下層ページに必須。

- 13px / muted色 / 最終項目のみ強調（font-weight: 600）
- SEO構造維持

# 5. フッターナビ設計

## カラム構造（最大4カラム）

- 事業 / 会社 / 法務 / お問い合わせ

## NAP表示（必須）

- 株式会社 清蓮 / 住所 / 電話番号

# 6. 情報の見せ方ルール

- 重要情報（予約制・電話番号・営業時間）はトップ上部に表示
- 大見出し最大6 / 長文は分割 / 箇条書き活用

# 7. SEO構造との整合

- h1は1ページ1つ / h2で大項目整理 / h3は補助
- 見出し飛び禁止

# 8. 禁止事項

- ナビ項目増殖 / メガメニュー過剰装飾 / 重要情報の深層化 / トリックUI

# 9. 成功基準

- 3秒以内に構造が理解できる / 電話が迷わず押せる / 会社の実在が明確 / 階層が自然

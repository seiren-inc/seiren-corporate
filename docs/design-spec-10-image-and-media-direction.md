# 清蓮コーポレートサイト デザイン設計書 10

## 画像 / メディアディレクション統合設計

---

# 1. 目的

写真・背景ビジュアル・質感演出・メディア最適化を統一する規格。
画像は装飾ではなく、信頼を視覚で証明する要素。

# 2. ビジュアル思想

- 静けさ / 光の柔らかさ / 余白 / 実在性 / 過剰でない
- Apple的「光と空気感」を基準とする

# 3. ブランドカラートーン

- 彩度は低め / コントラストは強すぎない / 青・緑系の自然光は可
- 強いビビッド色禁止

# 4. 画像カテゴリ設計

## Hero画像

- 内容：海の水平線・静かな波・朝夕の柔らかい光・霧がかった遠景
- 禁止：激しい波・ドローン派手映像・過度なHDR・人物感情訴求
- 仕様：16:9 / 1600px以上 / loading="eager" / 上に薄いグラデーション

## 会社紹介画像（実在証明）

- 受付・相談スペース・外観・看板
- 実写 / 明るい自然光 / 過度な演出なし

## 事業領域ビジュアル

- 抽象質感可（水面・石・光の反射・木目・柔らかい影）
- 広告的イラスト禁止

# 5. 画像配置ルール

- セクションごとに最大1枚
- width/height属性必須（CLS防止）
- Hero以外：loading="lazy" decoding="async"

# 6. picture要素必須

WebP対応を前提。`<picture>` + `<source srcset=".webp">` 構造。

# 7. alt設計ルール

- 禁止：「画像」「写真」
- 推奨：「海の水平線の写真」「清蓮の相談スペース」「戸塚町の店舗外観」

# 8. オーバーレイ設計

Heroにのみ使用可。

- rgba(0,0,0,0.25〜0.35)
- ノイズレイヤー: opacity 0.03程度
- 派手なフィルター禁止

# 9. モーションとの連携

画像Revealは以下のみ：

- opacity + translateY
- clip-path 横展開
- ズームイン禁止 / scale アニメ禁止

# 10. パフォーマンス要件

- WebP併用 / 200KB以下（Hero除く）/ 動画禁止 / 背景動画禁止

# 11. 禁止事項

- ストック臭い集合写真 / 彩度高い空 / 強いレンズフレア
- グラデーション乱用 / 3D装飾

# 12. 成功基準

- 写真が静か / 余白と調和 / 実在性が伝わる / ブランドカラーと喧嘩しない

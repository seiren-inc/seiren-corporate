# Freeze Prevention Check

> 対象: Antigravity / Codex / Claude Code
> 更新: 2026-04-14

---

## 概要

タスクが停滞・フリーズする前に自己チェックするためのワークフロー。

---

## チェックトリガー

以下のいずれかが発生したらこのチェックを実行：

- 同じ問題に2回以上つまずいている
- 次のアクションが不明確になった
- ループしていると感じる
- コンテキストが大きくなりすぎた

---

## チェックリスト

### 1. ゴール確認

```
[ ] 元のタスクのゴールを再確認したか
[ ] 現在の作業がゴールに向かっているか
[ ] スコープを超えた作業をしていないか
```

### 2. ブロッカー特定

```
[ ] 何が原因でつまずいているか特定できているか
[ ] 不明な前提条件はあるか
[ ] 外部依存（API・DB・env）の問題か、コード上の問題か
```

### 3. コンテキスト確認

```
[ ] コンテキストが大きすぎないか
[ ] 不要な情報を持ち込んでいないか
[ ] 必要な情報だけに絞れているか
```

### 4. エスカレーション判断

以下の場合は**即停止してユーザーに報告**：

```
- ゴールが不明確
- 禁止事項に接触する可能性がある
- スコープ外の変更が必要になった
- 2回以上同じエラーが発生
- 推測での実装が必要な状況
```

---

## 停止報告フォーマット

```markdown
## 停止報告

- 現在の状況:
- つまずいている原因:
- 試したこと:
- 必要な情報・判断:
- 推奨する次のアクション:
```

---

## 頻出フリーズポイント（共通）

| 状況 | 対処 |
|---|---|
| ゴールが曖昧 | AGENTS.md の Stop conditions を確認し停止 |
| env / secrets 不明 | `.env.example` を確認・不明時は停止 |
| 型エラーが解消しない | 型定義ファイルを先に確認 |
| ビルドが通らない | lint → typecheck → build の順で切り分け |
| コンテキスト肥大 | 無関係ファイルの読み込みを中止・必要最小限に絞る |
